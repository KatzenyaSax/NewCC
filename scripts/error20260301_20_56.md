# 依赖问题处理方案

## 问题描述

1. **冗余依赖声明**: `dafuweng-sales`、`dafuweng-finance`、`dafuweng-system`、`dafuweng-auth` 各自冗余声明了 `spring-boot-starter-web`、`mybatis-plus-boot-starter`、`redisson-spring-boot-starter`、`spring-boot-starter-validation`、`mysql-connector-j`。这些依赖已经通过 `common-core` 或 `common-mybatis` / `common-redis` 传递引入。
2. **版本硬编码**: `dafuweng-system` 的 common 依赖使用硬编码版本 `1.0.0` 而非 `${project.version}`。
3. **common-mybatis 冗余**: 自身依赖 `common-core`（已传递 `mybatis-plus-boot-starter`），又显式声明 `mybatis-plus-boot-starter`。
4. **common-redis 冗余**: 自身依赖 `common-core`（已传递 `redisson-spring-boot-starter`），又显式声明 `redisson-spring-boot-starter` 和 `spring-boot-starter-data-redis`。
5. **多模块聚合复杂**: `dafuweng-common` 拆成 4 个子模块但实际使用时各服务模块同时依赖 3 个（core/mybatis/redis），没有体现分层价值反而增加复杂度。

---

## 解决方案

**核心思路**: 将 `dafuweng-common` 从多子模块聚合简化为单一模块 `common-core`，服务模块只依赖这一个 common artifact。

### 第一步: 简化 dafuweng-common 为单一模块

删除 `dafuweng-common` 下的 3 个子模块目录:

```
dafuweng-common/
├── pom.xml          # 改为普通 jar，不再声明 <modules>
└── src/main/java/   # 原有 common-core 代码保留在这里
```

新的 `dafuweng-common/pom.xml` 继承 `dafuweng-parent`，直接声明所有 common 依赖:

```xml
<project>
    <parent>
        <groupId>com.dafuweng</groupId>
        <artifactId>dafuweng-parent</artifactId>
        <version>${project.version}</version>
        <relativePath>../pom.xml</relativePath>
    </parent>
    <artifactId>dafuweng-common</artifactId>
    <packaging>jar</packaging>

    <dependencies>
        <!-- Spring Boot Starter Web -->
        <dependency>
            <groupId>org.springframework.boot</groupId>
            <artifactId>spring-boot-starter-web</artifactId>
        </dependency>

        <!-- Spring Boot Starter Validation -->
        <dependency>
            <groupId>org.springframework.boot</groupId>
            <artifactId>spring-boot-starter-validation</artifactId>
        </dependency>

        <!-- MySQL -->
        <dependency>
            <groupId>com.mysql</groupId>
            <artifactId>mysql-connector-j</artifactId>
        </dependency>

        <!-- MyBatisPlus Boot Starter -->
        <dependency>
            <groupId>com.baomidou</groupId>
            <artifactId>mybatis-plus-boot-starter</artifactId>
        </dependency>

        <!-- Redisson -->
        <dependency>
            <groupId>org.redisson</groupId>
            <artifactId>redisson-spring-boot-starter</artifactId>
        </dependency>

        <!-- Lombok -->
        <dependency>
            <groupId>org.projectlombok</groupId>
            <artifactId>lombok</artifactId>
            <scope>provided</scope>
        </dependency>

        <!-- Hutool -->
        <dependency>
            <groupId>cn.hutool</groupId>
            <artifactId>hutool-all</artifactId>
            <version>5.8.26</version>
        </dependency>

        <!-- FastJSON2 -->
        <dependency>
            <groupId>com.alibaba.fastjson2</groupId>
            <artifactId>fastjson2</artifactId>
            <version>2.0.47</version>
        </dependency>

        <!-- MapStruct -->
        <dependency>
            <groupId>org.mapstruct</groupId>
            <artifactId>mapstruct</artifactId>
            <version>1.5.5.Final</version>
        </dependency>

        <dependency>
            <groupId>org.mapstruct</groupId>
            <artifactId>mapstruct-processor</artifactId>
            <version>1.5.5.Final</version>
            <scope>provided</scope>
        </dependency>

        <!-- Lombok MapStruct Binding -->
        <dependency>
            <groupId>org.projectlombok</groupId>
            <artifactId>lombok-mapstruct-binding</artifactId>
            <version>0.2.0</version>
            <scope>provided</scope>
        </dependency>
    </dependencies>

    <build>
        <plugins>
            <plugin>
                <groupId>org.apache.maven.plugins</groupId>
                <artifactId>maven-compiler-plugin</artifactId>
                <configuration>
                    <annotationProcessorPaths>
                        <path>
                            <groupId>org.projectlombok</groupId>
                            <artifactId>lombok</artifactId>
                            <version>${lombok.version}</version>
                        </path>
                        <path>
                            <groupId>org.projectlombok</groupId>
                            <artifactId>lombok-mapstruct-binding</artifactId>
                            <version>0.2.0</version>
                        </path>
                        <path>
                            <groupId>org.mapstruct</groupId>
                            <artifactId>mapstruct-processor</artifactId>
                            <version>1.5.5.Final</version>
                        </path>
                    </annotationProcessorPaths>
                </configuration>
            </plugin>
        </plugins>
    </build>
</project>
```

删除以下冗余子模块（目录）:

```
dafuweng-common/common-mybatis/
dafuweng-common/common-redis/
dafuweng-common/common-feign/          # 如果不需要也可删除
```

### 第二步: 迁移 common-feign（按需）

如果某些服务需要 Feign，将 `common-feign` 的内容迁移到 `dafuweng-common` 内，或保留为独立模块 `dafuweng-feign`（不依赖 common，直接依赖 `spring-cloud-starter-openfeign` + `spring-boot-starter`）。

建议: 如果只有 `dafuweng-auth` 使用 Feign，直接在 auth 的 pom 里声明 Feign 依赖，不单独建 common-feign 模块。

### 第三步: 简化服务模块 POM

每个服务模块（sales/finance/system/auth）的 pom.xml 改为:

```xml
<dependencies>
    <!-- dafuweng-common: 包含 web/validation/mybatis/redisson/lombok/hutool/fastjson2/mapstruct -->
    <dependency>
        <groupId>com.dafuweng</groupId>
        <artifactId>dafuweng-common</artifactId>
        <version>${project.version}</version>
    </dependency>

    <!-- 服务模块自身特有的依赖（JWT、OpenFeign 等）放在这里 -->
    ...
</dependencies>
```

删除所有冗余的以下依赖声明:

- `spring-boot-starter-web`
- `spring-boot-starter-validation`
- `mysql-connector-j`
- `mybatis-plus-boot-starter`
- `redisson-spring-boot-starter`

### 第四步: 修复 dafuweng-system 版本

将 `dafuweng-system/pom.xml` 中所有:

```xml
<version>1.0.0</version>
```

改为:

```xml
<version>${project.version}</version>
```

### 第五步: 删除 common 子模块聚合配置

`dafuweng-parent/pom.xml` 中的 `<modules>` 列表需要移除 `common-mybatis`、`common-redis`、`common-feign`（如果有单独列出）。

---

## 实施步骤

| 步骤 | 操作 | 影响范围 |
|------|------|---------|
| 1 | 删除 `dafuweng-common/common-mybatis/` 目录 | 无运行时影响 |
| 2 | 删除 `dafuweng-common/common-redis/` 目录 | 无运行时影响 |
| 3 | 删除 `dafuweng-common/common-feign/` 目录 | auth 模块需添加 feign 依赖 |
| 4 | 重写 `dafuweng-common/pom.xml` 为单一 jar | 核心变更 |
| 5 | 更新 `dafuweng-parent/pom.xml` 的 `<modules>` 列表 | 移除 3 个子模块 |
| 6 | 简化 `dafuweng-sales/pom.xml` | 移除 5 个冗余依赖 |
| 7 | 简化 `dafuweng-finance/pom.xml` | 移除 5 个冗余依赖 |
| 8 | 简化 `dafuweng-system/pom.xml` | 移除 5 个冗余依赖 + 修复版本 |
| 9 | 简化 `dafuweng-auth/pom.xml` | 移除 5 个冗余依赖 + 保留 JWT/Feign |

---

## 依赖传递关系（变更后）

```
dafuweng-parent
  └── dafuweng-common (单一 jar)
        ├── spring-boot-starter-web
        ├── spring-boot-starter-validation
        ├── mysql-connector-j
        ├── mybatis-plus-boot-starter
        ├── redisson-spring-boot-starter
        ├── lombok (provided)
        ├── hutool-all
        ├── fastjson2
        └── mapstruct + processor (provided)
  └── dafuweng-sales / dafuweng-finance / dafuweng-system / dafuweng-auth
        └── 仅依赖 dafuweng-common (一个 artifact)
```

---

## 验证方式

修改完成后在项目根目录执行:

```bash
mvn clean compile -DskipTests
```

所有模块应能一次性编译通过，无 missing dependency 错误。
